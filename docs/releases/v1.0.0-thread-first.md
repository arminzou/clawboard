## 0) Preconditions

- [ ] DB backup taken (SQLite file copy or volume snapshot)
- [ ] You know which environment you are changing (dev vs prod)

## 1) Migrations / Schema

- [ ] Start backend with migrations enabled (or run your migration script)
- [ ] Verify these tables exist:
  - [ ] `question_threads`
  - [ ] `thread_events`
  - [ ] `promotion_packets`
  - [ ] `thread_task_links`

## 2) Service Startup (Default-On)

- [ ] Restart backend
- [ ] Rebuild/restart frontend
- [ ] Verify routes are registered (sanity):
  - [ ] `GET /api/threads`
  - [ ] `GET /api/humans/:humanId/attention`
- [ ] Verify the **Attention** tab appears

## 3) Minimum Verification

### Automated

- [ ] `pnpm test:backend`
- [ ] (Optional) `pnpm test:frontend`
- [ ] `pnpm test:e2e` (if E2E is configured for your environment)

### Manual smoke flow (happy path)

- [ ] Create thread (open)
- [ ] Add clarification / work log events
- [ ] Transition: `open -> ready_to_plan -> pending_approval`
- [ ] Fill promotion packet
- [ ] Promote (must spawn >= 1 task)
- [ ] Confirm thread status is `promoted` and has `thread_task_links`
- [ ] Archive thread
- [ ] Clone archived thread; verify `cloned_from_thread_id` + `thread_cloned` events

## 4) Invariants to Re-check

- [ ] Promote enforces `spawned_task_count >= 1` (`400 promote_requires_task`)
- [ ] Mention payload validation (`400 mention_payload_invalid`)
- [ ] Objection event validation (`400 objection_payload_invalid`)
- [ ] Archived threads are terminal; resume via clone only

## 5) Rollback

- [ ] Restart services to recover from transient state
- [ ] If needed: restore DB from backup
- [ ] If needed: revert to pre-thread-first-default commit/tag
